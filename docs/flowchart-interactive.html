📋 Six-Stage Navigation Pipeline

The robot executes these stages in a continuous loop:

1️⃣
Sensor Acquisition

What: Read compass heading and GPS position simultaneously

Frequency: Every 200ms

Output: Raw heading, latitude, longitude, course

Hardware: QMC5883P + UBLOX GPS

2️⃣
Noise Filtering

What: Apply exponential moving average (EMA) to smooth data

Filter Coefficient: α = 0.1

Why: Removes sensor spikes, reduces jitter

Result: Clean, stable heading and position

3️⃣
Offset Learning

When: Robot moving > 3 km/h

How: GPS course = ground truth heading

Learning Rate: 5% per cycle

Benefit: Self-calibrating compass, no manual alignment

4️⃣
Target Calculation

Algorithm: Haversine formula for great-circle distance

Inputs: Current position, target waypoint coordinates

Outputs: Distance (meters), bearing (degrees)

Accuracy: ±2-5m depending on GPS signal

5️⃣
Heading Decision

Calculation: Error = Target Bearing - Current Heading

If |Error| > 25°: Turn in place (fast correction)

If |Error| ≤ 25°: Proportional steering (smooth turn)

Goal: Minimize heading error to target

6️⃣
Motor Control

Method: Differential drive (left + right motor speeds)

Cruise Speed: 50 (0-100 scale)

Turn Speed: 45 (0-100 scale)

Steering: Speed difference creates curves

📡 Real-Time Telemetry Output

Every cycle, the robot broadcasts this data via Bluetooth:

Example Output (one per second):
Hdg:045 GPS:042 Off:145 Err:-3 Dist:12.5
Hdg:046 GPS:041 Off:144 Err:-2 Dist:11.3
Hdg:047 GPS:040 Off:143 Err:-1 Dist:10.8
Hdg:048 GPS:039 Off:142 Err:+1 Dist:10.2
Field Range Meaning What It Tells You
Hdg (Heading) 0–359° Current compass direction Which way the robot is pointing
GPS 0–359° GPS course (calculated from movement) Which way the robot is actually moving
Off (Offset) -180 to +180 Learned boresight offset Compass correction factor (learned automatically)
Err (Error) -180 to +180 Heading error to target How far off course (negative = left, positive = right)
Dist (Distance) 0 to 9999m Distance to active waypoint How far until next waypoint reached

✅ Healthy Telemetry Signs

✓ Hdg slowly changing (smooth turns)

✓ Err reducing toward zero

✓ GPS tracking closely with Hdg

✓ Dist decreasing over time

✓ Off stable (not jumping)

⚠️ Warning Signs

✗ Hdg jumping erratically (bad compass)

✗ GPS frozen (GPS lock lost)

✗ Err oscillating wildly (motor issue)

✗ Off not converging (move faster)

✗ Dist stuck (encoder feedback needed)
